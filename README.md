# DS-Kentei-Study-Log (DS検定・G検定 学習記録)

データエンジニア（DE）へのキャリア転換を目指し、DS検定やG検定に向けた学習プロセスを記録するリポジトリです。

## 📌 コンセプト
「単なる暗記ではなく、実務に繋がる理解を」
NotebookLM等のAIツールを活用して知識を構造化し、自分の言葉で再構成した内容を蓄積しています。

## 📂 ディレクトリ構成
- `notebooks/`: 講義メモや解説、NotebookLMによる要約
- `sql/`: データ加工・抽出の実践クエリ集
- `excel/`: 統計計算やデータ集計の演習ファイル
- `terraform-aws/`: AWSインフラのコード管理（IaC）
- `logs/`: 学習における苦戦ポイントと解決策の記録

## 🚀 現在の目標
- **DS検定 リベンジ合格**: 2026年内の取得を目指し、弱点の統計学を重点的に強化。
- **G検定 合格**: 深層学習の理論と社会実装の基礎を習得。

## 🚀 プロジェクトのビジネス価値
本リポジトリで実装している技術スタックは、単なるスキルの羅列ではなく、以下の具体的な事業課題の解決を想定しています。

### 1. データクレンジングによる意思決定の安定化 (`notebooks/`)
- **課題**: センサー故障や入力ミスによる異常値が、売上予測や在庫管理の精度を著しく低下させていた。
- **解決**: 1.5xIQRルールに基づき、異常値を自動的に除去または境界値へ補正（Clipping）するモジュールを実装。
- **価値**: 予測のブレを最小化し、不要な在庫コストの削減と、欠品による機会損失リスクの低減に直接貢献します。

### 2. 堅牢なインフラによる分析環境の再現性 (`terraform-aws/`)
- **技術**: AWS / Terraform / Docker（順次実装中）
- **価値**: 「自分のPCでしか動かない」状態を排除。インフラをコード化（IaC）することで、チーム全体が同一の分析環境を即座に構築でき、分析プロジェクトの立ち上げスピードを最大化します。

## ☁️ AWS Infrastructure Automation with Terraform

Terraformを用いてAWS S3バケットを構築・管理するプロジェクトです。
実務を意識したリファクタリング、Git運用、およびトラブルシューティングを実践しました。

### 🛠 実装のこだわり
- **IaC (Infrastructure as Code)**: 手動操作を排除し、再現性の高いインフラ構築を実現。
- **Variablesの活用**: `variables.tf` による変数分離を実装し、保守性と再利用性を向上。
- **セキュリティ・マナーの徹底**: `.gitignore` を活用し、機密情報（tfstate）や不要なバイナリ（.terraform）を適切に管理。

### ⚡ トラブルシューティング（学びの記録）
本実装の過程で、以下の課題を自走して解決しました。
1. **GitHubのファイルサイズ制限への対応**: 
   - 誤ってGit管理に含めたプロバイダーの巨大バイナリ（約800MB）によるプッシュエラーを経験。
   - `git reset` および `.gitignore` の設定により、レポジトリをクリーンな状態に修正し、Git運用の作法を習得しました。
2. **Terraform最新構文への対応**:
   - 型指定（Type constraints）における非推奨な記述を、エラーログに基づき最新の記述（引用符の排除）へ修正。
